# 闪念备忘录 · 部署上线指南

> 面向零基础：把本地项目 `f:\beiwanglu` 变成别人能通过网址访问的在线网站。

---

## 一、先搞懂：部署到底是什么？

很多人会把两件事混在一起，其实不一样：

| 概念 | 是什么 | 咱们项目的情况 |
|------|--------|----------------|
| **部署网站** | 把**程序代码**放到服务器上，让别人能打开网页 | 要做的事 |
| **上传数据** | 把**备忘录内容**存到服务器数据库 | **目前没做** |

### 重要结论（必读）

咱们现在的备忘录数据存在浏览器 **localStorage** 里：

- 部署上线后，访客打开的是**同一个网站界面**
- 但每个人写的备忘，仍然只存在**他自己浏览器里**
- **不会**自动把你电脑里的备忘同步到服务器
- 换电脑、清缓存，数据就没了

所以：

> **部署 = 把网站程序挂到公网**  
> **≠ 把备忘数据上传到服务器**

如果以后想要「登录账号、多设备同步、数据在云端」，需要再加**后端 + 数据库**（这是下一阶段，文末有说明）。

## 三、推荐方案：Vercel 部署（最适合 Next.js，免费）

Vercel 是 Next.js 官方团队做的托管平台，对咱们这种项目最省事。

### 你需要准备

1. 一个 **GitHub 账号**（免费注册：https://github.com）
2. 本机安装 **Git**（免费下载：https://git-scm.com/download/win）
3. 一个 **Vercel 账号**（免费注册：https://vercel.com，可用 GitHub 登录）

> 你之前环境里没有 Git，部署前需要先装好 Git。

---

### 步骤 1：把代码上传到 GitHub

在项目目录打开终端（PowerShell）：

```bash
cd f:\beiwanglu

# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 第一次提交
git commit -m "闪念备忘录 MVP 初版"
```

然后去 GitHub 网站：

1. 点右上角 **+** → **New repository**
2. 仓库名填：`beiwanglu`（或你喜欢的名字）
3. 选 **Private**（私有）或 **Public**（公开）都行
4. **不要**勾选 "Add a README"（本地已有代码）
5. 创建后，按页面提示把本地仓库推上去，一般是：

```bash
git remote add origin https://github.com/你的用户名/beiwanglu.git
git branch -M main
git push -u origin main
```

第一次 push 可能会让你登录 GitHub（浏览器弹窗或输入 token）。

---

### 步骤 2：在 Vercel 导入项目

1. 登录 https://vercel.com
2. 点 **Add New…** → **Project**
3. 选择 **Import Git Repository**，找到刚上传的 `beiwanglu`
4. 配置页面一般**不用改**，保持默认即可：
   - **Framework Preset**：Next.js（自动识别）
   - **Build Command**：`npm run build`
   - **Output Directory**：留空（Next.js 自动处理）
   - **Install Command**：`npm install`
5. 点 **Deploy**

等待 1～3 分钟，完成后会给你一个网址，例如：

```
https://beiwanglu-xxxx.vercel.app
```

这就是你的**在线网站**，可以发给朋友访问。

---

### 步骤 3：以后改了代码怎么更新？

本地改完代码后：

```bash
git add .
git commit -m "描述你改了什么"
git push
```

Vercel 会**自动重新部署**，几分钟后线上网站就更新了。

---

### 可选：绑定自己的域名

如果你买了域名（如 `memo.example.com`）：

1. Vercel 项目 → **Settings** → **Domains**
2. 添加域名，按提示在域名服务商处添加 DNS 记录
3. 等待生效（通常几分钟到几小时）

## 五、备选方案 B：自己的云服务器（VPS）

适合：想完全自己掌控服务器（阿里云、腾讯云、华为云等）。

### 服务器要求

- 系统：Ubuntu 22.04 或类似 Linux
- 配置：1 核 2G 内存即可
- 已安装：**Node.js 20+**

### 大致流程

#### 1. 把代码传到服务器

可用 Git clone，或用 SFTP 工具（WinSCP、FileZilla）上传整个项目文件夹。

#### 2. 在服务器上安装依赖并构建

```bash
cd /path/to/beiwanglu
npm install
npm run build
```

#### 3. 用 PM2 保持网站常驻运行

```bash
npm install -g pm2
pm2 start npm --name "beiwanglu" -- start
pm2 save
pm2 startup
```

`npm start` 默认监听 **3000** 端口。

#### 4. 配置 Nginx 反向代理 + HTTPS（推荐）

让用户通过 80/443 端口访问，而不是直接 `:3000`。

Nginx 配置示例（域名换成你的）：

```nginx
server {
    listen 80;
    server_name memo.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

再用 **Certbot** 申请免费 HTTPS 证书：

```bash
sudo certbot --nginx -d memo.example.com
```

#### 5. 安全组 / 防火墙

云控制台放行：**80**、**443** 端口。

## 六、备选方案 C：Docker 部署（进阶）

如果熟悉 Docker，可以自建镜像。当前项目没有自带 Dockerfile，需要新建（示例）：

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
COPY --from=builder /app/package*.json ./
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/public ./public
EXPOSE 3000
CMD ["npm", "start"]
```

构建并运行：

```bash
docker build -t beiwanglu .
docker run -d -p 3000:3000 --name beiwanglu beiwanglu
```

## 七、部署时常见问题

### Q1：`npm run build` 失败怎么办？

- 先在本地修好，能 build 成功再部署
- 把完整报错复制给 AI 或搜索引擎排查

### Q2：部署成功但打开是空白页？

- 打开浏览器开发者工具（F12）看 Console 有没有报错
- 确认 Vercel 的 Node.js 版本 ≥ 20（项目 Settings → General → Node.js Version）

### Q3：我电脑里的备忘，上线后还在吗？

- **在你自己电脑的浏览器里**：如果还是访问 localhost，localStorage 数据还在
- **访问线上网址**：是**全新空白**的，因为线上域名和 localhost 的 localStorage 不共享

### Q4：想让所有人共用一份数据，或登录后看到自己的数据？

需要改造架构，增加：

```
浏览器 → API 接口（后端）→ 数据库（PostgreSQL / SQLite 等）
```

可选技术栈：

- **Next.js API Routes** + Prisma + PostgreSQL（继续用 Next.js 一把梭）
- 或参考 Memos：**Go 后端** + SQLite

这是「第二期」，不是当前 MVP 部署的必需步骤。

### Q5：部署要花多少钱？

| 方案 | 费用 |
|------|------|
| Vercel 免费版 | 个人小项目通常够用，¥0 |
| 云服务器 VPS | 约 ¥30～100/月 |
| 自定义域名 | 约 ¥30～80/年 |

## 十二、总结

1. **部署**是把网站程序放到公网服务器，不是把备忘数据上传上去。  
2. 咱们项目**只有前端**，用 **Vercel** 部署最简单。  
3. 当前数据在 **localStorage**，每个访问者各自一份，不互通。  
4. 想云端存数据，需要下一步加**后端和数据库**。
